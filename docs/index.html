CrossDoc

The documentation management tool that re-defines how and when you can interact with your comments. Providing features such as comment classification and comment scope, you can expand the descriptive capabilities of your project's documentation.

The Program

CrossDoc is a command line tool and text-editor plugin that allows you to seperate your comments from your code. This separation means that CrossDoc can provide unique commenting features previously impossible. Features such as comment classification and comment scope support.

CrossDoc is an open source project and is available on GitHub

Comment Classification

Comment classification, or comment sets, is the grouping of similar comments so that developers can toggle through distinct comment sets. For example, a team may have a group of comments classified as Newbie Notes that includes verbose comments describing all aspects of the functions it encompasses. An experienced developer may be more interested in the classification labeled Design Rationale that includes comments describing why a function works as it does, rather than how.

Comment Scope

Comment scope is the ability to declare both a starting and ending point for a particular comment. In traditional commenting, a comment simply describes an arbitrary number of lines of code below the comment, but with CrossDoc, the addition of scope greatly broadens the expressive power of in-file comments.
